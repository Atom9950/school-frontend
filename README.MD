## ğŸŒŸ Overview

<div align="center">

![Overview Banner](./assets/images/overview-banner.png)

</div>

The School Management System is a robust, full-stack web application designed to digitize and automate the complex administrative processes within educational institutions. This system provides a centralized platform for managing departments, subjects, classes, teachers, students, and attendance tracking with an intuitive admin dashboard.

Built with scalability and performance in mind, the system leverages the PERN stack (PostgreSQL, Express, React, Node.js) combined with modern UI frameworks and security best practices to deliver a seamless user experience.

### Key Highlights

- **ğŸ¯ Role-Based Access Control**: Secure admin dashboard with granular permissions

- **ğŸ”’ Enterprise Security**: Powered by Arcjet for advanced security features

### Prerequisites

Before you begin, ensure you have the following installed:

- **Storage**: At least 2GB free disk space
- **OS**: Windows 10+, macOS 10.15+, or Linux (Ubuntu 20.04+)

## ğŸ“¦ Installation

### Frontend Setup

1. **Clone the frontend repository**
```bash
git clone https://github.com/Atom9950/school-frontend.git
cd school-frontend
```

2. **Install dependencies**
```bash
npm install
# or
yarn install
```

3. **Create environment file**
```bash
cp .env.example .env
```

4. **Configure environment variables**
```env
VITE_API_URL=http://localhost:5000/api
VITE_APP_NAME=School Management System
```

5. **Start development server**
```bash
npm run dev
# or
yarn dev
```

The application will be available at `http://localhost:5173`

### Backend Setup

1. **Clone the backend repository**
```bash
git clone https://github.com/Atom9950/school-backend.git
cd school-backend
```

2. **Install dependencies**
```bash
npm install
# or
yarn install
```

3. **Create environment file**
```bash
cp .env.example .env
```

4. **Configure environment variables**
```env
# Database Configuration
DATABASE_URL=postgresql://username:password@localhost:5432/school_db

# Server Configuration
PORT=5000
NODE_ENV=development

# Security
JWT_SECRET=your-secret-key-here
ARCJET_KEY=your-arcjet-api-key

# Monitoring
SITE24X7_API_KEY=your-site24x7-key
```

5. **Set up the database**
```bash
# Create database
createdb school_db

# Run migrations
npm run db:migrate
# or
yarn db:migrate

# Seed initial data (optional)
npm run db:seed
# or
yarn db:seed
```

6. **Start the server**
```bash
npm run dev
# or
yarn dev
```

The API will be available at `http://localhost:5000`

### Docker Setup (Alternative)

If you prefer using Docker:

```bash
# Clone both repositories
git clone https://github.com/Atom9950/school-frontend.git
git clone https://github.com/Atom9950/school-backend.git

# Create docker-compose.yml in project root
docker-compose up -d
```

## âš™ï¸ Configuration

### Frontend Configuration

**vite.config.ts**
```typescript
import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';
import path from 'path';

export default defineConfig({
  plugins: [react()],
  resolve: {
    alias: {
      '@': path.resolve(__dirname, './src'),
    },
  },
  server: {
    port: 5173,
    proxy: {
      '/api': {
        target: 'http://localhost:5000',
        changeOrigin: true,
      },
    },
  },
});
```

### Backend Configuration

**Database Configuration (drizzle.config.ts)**
```typescript
import type { Config } from 'drizzle-kit';

export default {
  schema: './src/db/schema.ts',
  out: './drizzle',
  driver: 'pg',
  dbCredentials: {
    connectionString: process.env.DATABASE_URL!,
  },
} satisfies Config;
```

### Security Configuration (Arcjet)

```typescript
import arcjet, { detectBot, shield } from '@arcjet/node';

const aj = arcjet({
  key: process.env.ARCJET_KEY!,
  rules: [
    shield({ mode: 'LIVE' }),
    detectBot({
      mode: 'LIVE',
      allow: [],
    }),
  ],
});
```

### Security Features

   - Bot detection and mitigation

   - Role-based access control (RBAC)
   - Secure password hashing
   - Session management

   - SQL injection prevention (via ORM)
   - XSS protection
   - CSRF tokens

4. **API Security**
   - CORS configuration
   - Request signing
   - API key authentication
   - Encrypted data transmission (HTTPS)

### Best Practices

- Keep dependencies updated
- Use environment variables for sensitive data
- Regular security audits
- Implement logging and monitoring
- Follow OWASP guidelines
